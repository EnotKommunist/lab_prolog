**Задание 1**

**Задача 1**

**Текст задачи**

Составить программу нахождения числа, которое образуется из данного
натурального числа при записи его цифр в обратном порядке. Например, для
числа 1234 получаем ответ 4321. 

**Алгоритм решения**

Предикат reverse_number/2 запускает рекурсивный процесс, обнуляя аккумулятор. На каждом шаге reverse_number_acc/3 извлекает последнюю цифру числа (LastDigit = N mod 10) и отсекает её (N1 = N // 10). Накопленное значение формируется путём сдвига предыдущего результата влево и добавления новой цифры (NewAcc = Acc * 10 + LastDigit). Рекурсия продолжается, пока исходное число не станет равным нулю. В терминальном случае аккумулятор, содержащий перевёрнутое число, возвращается как итоговый результат. Предикат start обеспечивает ввод числа с клавиатуры, вызов реверса и вывод полученного значения.

**Задание 2**

**Задача 1**

**Текст задачи**

В заданном списке определить максимальное количество подряд идущих
положительных чисел. 

**Алгоритм решения**

Предикат max_consecutive_positive/2 запускает рекурсивный процесс с двумя аккумуляторами: текущая длина последовательности и глобальный максимум, оба изначально равны нулю. При обходе списка если текущий элемент положительный, то текущий счетчик увеличивается на единицу, а глобальный максимум остается неизменным. Если же элемент не положительный, то текущая последовательность прерывается: глобальный максимум обновляется максимальным значением между текущим счетчиком и предыдущим глобальным максимумом, а текущий счетчик сбрасывается в ноль. Когда список заканчивается, итоговым результатом становится максимальное значение из оставшегося текущего счетчика и глобального максимума. Предикат start обеспечивает ввод списка с клавиатуры, вызов основного предиката и вывод результата.

**Задание 3**

**Задача 1**

**Текст задачи**

Определим множество как список без повторяющихся элементов. Найти
дополнение множества. 

**Алгоритм решения**

Нахождения дополнения множества A относительно универсума U. Предикат complement/3 вызывает subtract/3, который последовательно обходит элементы универсума U. Для каждого элемента проверяется, входит ли он в множество A с помощью member/2. Если элемент принадлежит A, он отсекается с помощью оператора отсечения и не попадает в результирующий список, а рекурсия продолжается с оставшейся частью универсума. Если элемент не принадлежит A, он добавляется в начало результирующего списка, после чего рекурсивно обрабатывается хвост универсума. Когда универсум заканчивается, результирующий список представляет собой все элементы U, которых нет в A, то есть дополнение множества. Предикат start обеспечивает ввод универсума и множества A с клавиатуры, вызов complement/3 и вывод всех трех множеств для наглядности.

**Задание 4**

**Задача 1**

**Текст задачи**

Брауну, Джонсу и Смиту предъявлено обвинение в соучастии в ограблении
банка. Похитители скрылись на поджидавшем их автомобиле. На следствии
Браун показал, что преступники скрылись на синем «Бьюике»; Джонс сказал,
что это был черный «Крайслер», а Смит утверждал, что это был «Форд Мустанг»
и ни в коем случае не синий. Стало известно, что, желая запутать следствие,
каждый из них указал правильно либо только марку машины, либо только ее
цвет. Какого цвета и какой марки был автомобиль? 

**Алгоритм решения**

Программа начинается с определения фактов: возможные цвета автомобиля — blue, black, red, а возможные марки — buick, chrysler, ford. Основной предикат solve/2 перебирает все комбинации цвета и марки через вызовы color(Color) и make(Make), а затем проверяет условия, заданные свидетелями. Условие для Брауна требует, чтобы ровно одно из двух утверждений было истинным: либо автомобиль синий, но не Бьюик, либо автомобиль не синий, но Бьюик. Условие для Джонса аналогично требует, чтобы ровно одно было верным: либо автомобиль черный, но не Крайслер, либо автомобиль не черный, но Крайслер. Условие для Смита требует, чтобы ровно одно утверждение было истинным: либо автомобиль Форд и не синий, либо автомобиль не Форд и синий. Таким образом, solve/2 возвращает только те комбинации цвета и марки, которые удовлетворяют всем трем условиям одновременно. Предикат start запрашивает у пользователя ввод цвета и марки, затем проверяет, соответствует ли введенная комбинация решению задачи, и выводит соответствующее сообщение.
